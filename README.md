## BiliBili 直播间自动点赞

因为B站主播一段时间不看牌子就会自动熄灭,如果粉丝牌过多的话就(~~~DD斩首!~~~)

所以写了小工具用来避免粉丝牌熄灭,目前的策略是通过给主播点赞来点亮灯牌(但需要观察主播是否在开播,建议找尽可能多主播开播的时机来运行())

## 实现

基于 Node.js + Playwright 模仿人类进行直播间的点赞操作

## 核心特性 

1. **配置与脚本分离**：直播间号/点赞规则写在`live_number.json`，增删改无需动主脚本 
2. **Cookie免登**：一次登录生成Cookie，后续运行直接免登，无需重复扫码 
3. **防风控设计**：单页面依次跳转、自定义点赞间隔，模拟真人操作 
4. **可视化执行**：Firefox浏览器窗口直接扫码/运行，操作过程可见可验证 
5. **异常容错**：单个直播间处理失败不中断整体流程，控制台打印详细执行日志 

## 环境准备

### 安装 Node.js

```
# ArchLinux
sudo pacman -S nodejs npm -y
# Debian / Ubuntu
sudo apt update && sudo apt install nodejs npm -y
```

## 快速配置 & 使用

### 步骤 1：项目初始化

```
# 进入项目根目录
cd 你的项目根目录
# 安装依赖
npm install
```

### 步骤 2：编辑直播间配置（核心！必做）

打开项目根目录的 `live_number.json` 文件，按**指定格式**添加需要点赞的直播间，示例如下：

```
[
  {"liveNum": 27183290, "likeCount": 35, "likeDelay": 600},
  {"liveNum": 123456, "likeCount": 20, "likeDelay": 500}
]
```

### 步骤 3：运行脚本

```
# 首次使用/Cookie 过期时：执行登录脚本，生成 bilibili_cookies.json 免登文件
node bilibili_login.js
# 已登录（有 bilibili_cookies.json 文件）：直接执行点赞脚本
node likbtn.js
```
